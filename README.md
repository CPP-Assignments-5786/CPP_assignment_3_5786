# מטלה שלישית - קומפוזיציה, בנאים, מפרקים ומערכים דינמיים

## יושרה אקדמית

במהלך העבודה על המטלות, מותר להתייעץ עם סטודנטים אחרים ולחפש מידע באינטרנט. עם זאת, חל איסור להעתיק קטעי קוד שלמים ממקורות חיצוניים, כולל סטודנטים אחרים, אתרי אינטרנט ומודלי בינה מלאכותית (כגון ChatGPT).

יש לדווח על כל עזרה שקיבלתם, בין אם מדובר בהתייעצות עם סטודנטים אחרים או במידע שנמצא באינטרנט, בהתאם ל[תקנון היושר של המחלקה](https://www.ariel.ac.il/wp/cs/wp-content/uploads/sites/88/2020/08/Guidelines-for-Academic-Integrity.pdf). **במקרה של שימוש בכלי בינה מלאכותית (AI), יש לצרף את הפרומפטים שהוזנו ואת התשובות שהתקבלו**.

## הקדמה

מטלה זו היא חלק מהקורס "C++ תכנות מתקדם" ומטרתה להדגים מושגים מתקדמים בשפת C++ באמצעות מימוש מערכת ניהול ספריה. המטלה מתמקדת בקומפוזיציה בין מחלקות, ניהול מחזור חיים של אובייקטים (בנאים ומפרקים), ניהול מערכים דינמיים, ופונקציות חברות.

## מבנה המטלה

במטלה זו תתבקשו להשלים קוד חסר בקבצי המימוש (`.cpp`) עבור שתי מחלקות מרכזיות: `Book` ו-`LibraryCard`. המחלקות מדגימות יחסי קומפוזיציה — `Book` מכיל אובייקט `Author`, ו-`LibraryCard` מכיל מערך דינמי של מצביעים ל-`Book`.

## דרישות והנחיות

- להשלים את הקוד החסר בקבצי המימוש `Book.cpp` ו-`LibraryCard.cpp`
- לוודא שהקוד מתקמפל ופועל ללא שגיאות
- לממש ניהול נכון של זיכרון דינמי (הקצאה ושחרור)
- להדגים שימוש נכון בכל מושגי C++ הנדרשים
- לעבור בהצלחה את הטסטים הכתובים בקובץ `test.cpp`

## תכונות C++ שיש להדגים

1. **קומפוזיציה (Composition)** — מחלקת `Book` מכילה אובייקט `Author`; מחלקת `LibraryCard` מכילה מערך דינמי של מצביעים ל-`Book`
2. **בנאים ומפרקים (Constructors & Destructors)** — בנאי ברירת מחדל, בנאי עם פרמטרים, בנאי העתקה ומפרק
3. **מחיקת מערכים (Destruct-arrays)** — שימוש נכון ב-`delete[]` למחיקת מערכים דינמיים
4. **פרמטרים לפי הפניה (Reference parameters)** — העברת אובייקטים לפונקציות באמצעות `const&`
5. **החזרת הפניה (Reference-return)** — פונקציות המחזירות הפניה לאובייקט פנימי
6. **מתודות קבועות (Const methods)** — מתודות שאינן משנות את מצב האובייקט
7. **פונקציות חברות (Friend functions)** — פונקציות עם גישה לאיברים הפרטיים
8. **עומס-יתר של פונקציות (Function overloading)** — מספר גרסאות לאותה פונקציה עם פרמטרים שונים
9. **מימוש Inline ו-Outline** — מתודות קצרות (getters) ממומשות בתוך קובץ ה-header; מתודות ארוכות ממומשות בקובץ ה-`.cpp`
10. **איברים סטטיים (Static members)** — מונה כולל של ספרים וכרטיסיות שנוצרו
11. **בדיקות יחידה (Unit Testing)** — שימוש בספריית `doctest` לבדיקות אוטומטיות

## קבצים במטלה

### קבצים שסופקו:

- **Book.hpp** — קובץ הכרזה למחלקת `Book` ולמבנה `Author`. מכיל הגדרות מלאות של ה-API, הסברים מפורטים לסטודנטים ומימושי Inline של ה-getters
- **LibraryCard.hpp** — קובץ הכרזה למחלקת `LibraryCard` המנהלת רשימת ספרים שאולים. מכיל הגדרות מלאות של ה-API ומימושי Inline של ה-getters
- **main.cpp** — תוכנית ראשית המדגימה שימוש בכל המושגים הנדרשים עם הערות המציינות את הפלט הצפוי
- **test.cpp** — קובץ בדיקות יחידה המכסה את כל פונקציונליות המחלקות
- **doctest.h** — ספרייה לביצוע בדיקות אוטומטיות
- **Makefile** — קובץ לקימפול והרצה אוטומטית של הפרויקט

### קבצים שעליכם ליצור ולממש:

- **Book.cpp** — מימוש מחלקת `Book`: בנאים, מפרק, מתודות getter ו-setter, מתודות עזר (`isLargeBook`, `getCategory`, `getInfo`, `print`) ופונקציות חברות (`compareByPages`, `haveSameAuthor`)
- **LibraryCard.cpp** — מימוש מחלקת `LibraryCard`: בנאים (כולל בנאי העתקה עמוקה), מפרק עם שחרור נכון של המערך הדינמי, ניהול רשימת הספרים השאולים (`borrowBook`, `returnBook`, `findBorrowedBook`) ופונקציית חבר (`compareByBorrowCount`)

## הוראות קימפול והרצה

### שימוש ב-Makefile:

הפרויקט כולל `Makefile` שמאפשר קימפול והרצה בקלות:

```bash
make        # מקמפל ומריץ את התוכנית הראשית
make test   # מקמפל ומריץ את בדיקות היחידה
make clean  # מנקה את כל קבצי הקימפול
```
